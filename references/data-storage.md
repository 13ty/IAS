# IAS Data Storage Reference

## Directory Layout

```
IAS/
├── data/                          # Primary data store (JSON + CSV)
│   ├── tool_registry.json         # ⭐ Core: 16 tools, 6 domains
│   ├── taxonomy_mapping.json      # 4D classification (Layer/Function/Domain/Pattern)
│   ├── proposals.csv              # Tool proposals queue (URLs await audit)
│   └── audits/                    # Generated audit reports (.md files)
│
├── backend/                       # Runtime config (NOT in VCS)
│   ├── .planCS                    # Scaffold map — directory structure
│   ├── .envy                      # Machine paths, ports, services
│   └── settings.yaml              # Pipeline settings
│
├── state/                         # Pipeline state (ephemeral)
│   ├── schema.json                # JSON Schema for state validation
│   └── tech_stack_oracle.json     # Current pipeline state (generated)
│
└── references/                    # Documentation (markdown)
    ├── data-storage.md            # This file
    ├── taxonomy-framework.md      # 4D taxonomy design
    ├── acceptance-criteria.md     # Test criteria
    ├── scoring-rubrics.md         # Scoring methodology
    ├── methodology-database.md    # Research methodology
    └── tool-integrations.md       # Integration patterns
```

## Data Formats

### 1. `data/tool_registry.json` — Core Registry

**Format**: JSON with nested structure

```json
{
  "version": "2.0.0",
  "last_updated": "2026-06-07",
  "tools": {
    "ToolName": {
      "name": "Display Name",
      "version": "1.0.0",
      "type": "cli|python|jar|plugin|...",
      "domain": "visual|code-analysis|data|dev|devops|ai",
      "priority": "hot|warm|cold",
      "status": "active|trial|assess|hold",
      "location": "R:\\Dev\\Tools\\ToolName",
      "run_command": "tool --flag input",
      "update_command": "choco upgrade tool",
      "alternatives": ["Alt1", "Alt2"],
      "description": "What it does"
    }
  },
  "domains": {
    "visual": ["Mermaid-CLI", "D2", "Graphviz", "PlantUML"],
    "code-analysis": ["TreeSitter", "Semgrep", "cloc"]
  }
}
```

**Fields**:

- `location` — Absolute path to tool installation (Windows paths)
- `run_command` — How to invoke (may contain port numbers for services)
- `alternatives` — Used by heatmap overlap detection

### 2. `data/proposals.csv` — Proposal Queue

**Format**: CSV with headers

```csv
timestamp,url,source,type,notes,status
2026-06-07T15:36:31+00:00,https://github.com/pytest-dev/pytest,user,repo,Testing framework,audited
```

**Status lifecycle**: `pending` → `audited` → `adopted|rejected|pilot`

**URL patterns stored**:

- GitHub repos: `https://github.com/user/repo`
- PyPI packages: `https://pypi.org/project/name`
- npm packages: `https://www.npmjs.com/package/name`
- Docker images: `https://hub.docker.com/r/name`
- VS Code extensions: `https://marketplace.visualstudio.com/items?itemName=...`

### 3. `data/audits/*.md` — Audit Reports

**Format**: Markdown with YAML frontmatter (generated by `proposal_audit.py`)

```markdown
---
type: repo-analysis
version: 3.0.0
created: 2026-06-12
category: skill-2-benchmark
tags:
  - "@analysis"
  - "@benchmark"
---

# 🔬 Analiza Repozytorium: user/repo

...
```

**Naming**: `{date}_{slugified-name}.md` (e.g., `20260612_pytest-dev-pytest.md`)

### 4. `data/taxonomy_mapping.json` — 4D Classification

**Format**: JSON mapping tool IDs to 4-axis coordinates

```json
{
  "Mermaid-CLI": {
    "layer": "A8",
    "function": "B5",
    "domain": "C6",
    "pattern": "D12"
  }
}
```

**Axes**:

- **Layer** (A0-A8): Physical Storage → Application
- **Function** (B0-B6): Ingest → Visualization
- **Domain** (C0-C9): System/Infra → AI/ML
- **Pattern** (D0-D12): CLI → Notebook

### 5. `backend/.envy` — Machine Config

**Format**: YAML-like (simple key=value with sections)

```yaml
_meta:
  machine: DESKTOP
  os: win32

runtime:
  python: python
  gh: gh

paths:
  ias_root: C:\Users\13ty\.config\opencode\plugins\IAS
  vaults: C:\Users\13ty\.config\opencode\Vaults

services:
  lmstudio: http://localhost:1234
```

**NOT in version control** — machine-specific paths.

### 6. `state/tech_stack_oracle.json` — Pipeline State

**Format**: JSON (validated against `state/schema.json`)

```json
{
  "version": "1.0.0",
  "lastUpdated": "2026-06-12T12:00:00Z",
  "currentPhase": "IDLE|INVENTORY|ANALYZER|SCOUT|COMPLETE",
  "projectPath": "R:\\Dev\\MyProject",
  "inventory": { "..." },
  "analysis": { "..." },
  "scout": { "..." },
  "decisions": [],
  "history": []
}
```

**Ephemeral** — regenerated each pipeline run.

## URL Storage: Where Do Repository URLs Live?

### 1. Proposals Queue (Input)

**File**: `data/proposals.csv`  
**Added by**: `proposal_collect.py` or `/tool-use` command

```
/ias-audit https://github.com/user/repo "optional notes"
```

**URL stored in**: `url` column of CSV  
**Status**: `pending` until audited

### 2. Tool Registry (Registered Tools)

**File**: `data/tool_registry.json`  
**Added by**: `tool_register.py` after successful audit

```json
{
  "location": "R:\\Dev\\Tools\\ToolName",
  "run_command": "tool --flag",
  "update_command": "choco upgrade tool"
}
```

**Note**: Registry stores **local paths**, not remote URLs. Remote URLs are only in proposals.

### 3. Audit Reports (Historical)

**File**: `data/audits/{date}_{name}.md`  
**Generated by**: `proposal_audit.py`

Contains the original URL in the markdown body:

```markdown
## 📍 Źródło

- URL: https://github.com/user/repo
- Typ: repo
- Status: audited
```

### 4. Pipeline State (Transient)

**File**: `state/tech_stack_oracle.json`  
**Written by**: `orchestrator.py` during pipeline runs

```json
{
  "scout": {
    "results": [
      {
        "repo": "user/repo",
        "url": "https://github.com/user/repo",
        "scorecard": 8.5,
        "final_score": 0.91
      }
    ]
  }
}
```

**URLs here are ephemeral** — overwritten each run.

### 5. MCP Server (Runtime)

**File**: `scripts/mcp_server.py`  
**Exposed via**: `tool_registry_list` and `tool_details` tools

Returns `location` field (local paths), not remote URLs.

## URL Flow Diagram

```
User adds URL
     │
     ▼
proposal_collect.py ──→ data/proposals.csv (pending)
     │
     ▼
proposal_audit.py ──→ data/audits/*.md (historical record)
     │
     ▼
tool_register.py ──→ data/tool_registry.json (local path only)
     │
     ▼
orchestrator.py ──→ state/tech_stack_oracle.json (transient)
```

## Quick Reference: "Where Do I Find X?"

| Data                  | File                           | Format   |
| --------------------- | ------------------------------ | -------- |
| All tools             | `data/tool_registry.json`      | JSON     |
| Tool URLs (proposals) | `data/proposals.csv`           | CSV      |
| Audit history         | `data/audits/*.md`             | Markdown |
| 4D classification     | `data/taxonomy_mapping.json`   | JSON     |
| Machine paths         | `backend/.envy`                | YAML     |
| Pipeline state        | `state/tech_stack_oracle.json` | JSON     |
| Scaffold map          | `backend/.planCS`              | YAML     |

---

## Notes

- **No remote URLs in registry** — `tool_registry.json` only stores local `location` paths
- **Proposals are the URL store** — `proposals.csv` is the single source of truth for repository URLs
- **Audits are permanent** — `data/audits/` files are never deleted (historical record)
- **State is ephemeral** — `state/tech_stack_oracle.json` is overwritten each pipeline run
- **`.envy` is local** — never commit; machine-specific paths
