## Decyzje i Rekomendacje

### 1. `/ias-query` jako P0 🟡

**Dlaczego teraz:**

- Core feature IAS, mało roboty, jasny plan
- `lancedb_upsert.py` i `iamsparta/semantic/search.py` mają już 90% kodu
- Wystarczy: `ias_query.py` + `ias_reindex.py` + komenda w `plugin.json`
- ~150 linii nowego kodu, nie piszemy od zera

**Co robi:**

- User types `/ias-query "diagram as code"`
- Embedding query → LanceDB vector search
- Format output: name | domain | path | run_command | score
- Optional: pipe to fzf for interactive selection

**Plik do edycji:**

- `commands/ias-query.md` (dokumentacja komendy slash)
- `scripts/ias_query.py` (główny skrypt)
- `scripts/ias_reindex.py` (batch re-index)
- `plugin.json` (dodaj komendę)
- `mcp_server.py` (dodaj narzędzie MCP)

### 2. `enrich_repo_data.py` jako P1 🔲

**Dlaczego później:**

- Pipeline quality, ale wymaga zgłębienia w `repo_card.json` i `repo_utils.py`
- Nie jest w priorytetach refocusu
- Może być osobny feature (nie core IAS)

**Co robi:**

- PyGithub → metadane (stars, topics, license)
